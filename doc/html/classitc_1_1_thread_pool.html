Detailed Description

This class is a container for taks queues and it does not manage the threads. It is capable to create and invoke threads on task enqueu, however after the task is done, the thread perists in mActiveThreads container and will not be moved to mPassiveThreads queue. You does not need this class without a itc::ThreadPoolManager. So instantiate one, which will create an instance of itc::ThreadPool within itself and will effectively manage the threads in a pool.
