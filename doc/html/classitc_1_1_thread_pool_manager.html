Detailed Description

This class manages an instance of the itc::ThreadPool class. The default behavior is to expand threads within a itc::ThreadPool instance logarithmycally based on current value of mMaxThreads attribute of the itc::ThreadPool if the current value of the attibute mTaskQueue of the itc::ThreadPool is 25% longer then amount of threads in itc::ThreadPool. The upper limit of number of threads in itc::ThreadPool is defined by itc::ThreadPoolManager.mMaxThreads + itc::ThreadPoolManager.mOvercommitThreads. Right now this class is too simple for soft and hard upper limit, which will be changed later.

Need to add functionality to manage thread pool on time-line and resource usage.
